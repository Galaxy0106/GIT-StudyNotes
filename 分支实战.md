**分支切换会改变工作目录中的文件**

切换分支：

    每次切换分支前保证当前分支一定是干净的，即已提交状态
    
    修改的内容：
        HEAD
        暂存区
        工作目录
    
    坑：
        切换分支时当前分支有未提交的暂存或是未暂存的修改
        分支虽然可以切换成功
        但是却会污染其他分支  

快进合并：
    git merge branch_name
快进合并不会产生冲突

典型合并：
存在冲突

## 解决冲突
```shell
modify
git add ./
git commit -m ""
git branch -d ...
```
