# GIT学习

## 基本概念

### 版本控制

**集中化的版本控制**

SVN

```
具有一个单一的集中管理的服务器
优点：
一定程度上可以看到别人的代码，便于协同工作
管理相对容易
缺点：
中央服务器的单点故障
```

**分布式的版本控制**

GIT

不只保存项目的最新快照，而是保存整个代码仓库的完整镜像

存放的是索引

```
每个客户端可以放下整个项目的历史记录
版本控制是在本地进行的
Github挂了并不影响，因为Github也相当于是一个客户端
```





## 简介&安装

推荐使用git bash

### git配置

```shell
git config --list
git config --global user.name "..."
git config --global user.email ...

--system	整个系统
--global	单个用户
空			单个项目
```
